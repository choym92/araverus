Concept A: Hub Cards
Parent groups are elevated to visual cards with sub-thread pills inside. Each parent card is a "hub" showing all angles of a macro story at a glance. Standalone threads live below in a compact list.
Key idea: Parent card = "this is one big story." Sub-thread pills = "here are the angles." User scans 7 cards instead of scrolling 80 threads.

Concept B: Nested Accordion
Evolves the current accordion into a two-level hierarchy. Parent groups are collapsible sections containing sub-thread rows, which themselves expand to show articles. Familiar file-explorer pattern.
Key idea: Minimal visual change from current design. Just wrap existing thread rows inside parent sections. Lowest effort, still communicates hierarchy.

Concept C: Thread Detail Page
Clicking a parent card navigates to /news/thread/[id]. Shows parent overview, horizontal sub-thread selector, and a combined timeline of all articles across sub-threads.
Key idea: This is the "landing page" for a macro story. Everything about US-Iran in one view. Sub-thread chips act as filters. Timeline shows the full narrative arc.

Concept D: Visual Cards with Hero
Each parent group gets a rich visual card with gradient hero, sub-thread chips, and latest headline. More editorial/magazine feel. Designed for visual impact.
Key idea: Make Stories tab feel premium. Gradient heroes give each macro story a distinct visual identity. Sub-threads as pills below. Good for "wow factor" but heavier.

Recommended approach:

Phase 1: Concept A (Hub Cards) for Stories tab list view — best balance of intuitiveness, info density, and effort.

Phase 2: Concept C (Thread Detail Page) as the click-through destination — gives the full narrative arc.

Phase 3: Add Concept D (Visual Cards) hero treatment to the top 1-2 parent stories for visual impact.

Concept B (Nested Accordion) is a safe fallback if you want to ship fast with minimal visual change.
