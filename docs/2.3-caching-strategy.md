# Caching Strategy

## Architecture

```
CDN (Vercel Edge) → ISR (Next.js) → unstable_cache → Supabase
```

- **CDN**: `Cache-Control` headers for API routes (`/api/news`, `/rss.xml`)
- **ISR**: `export const revalidate` on page routes — stale pages served while regenerating
- **unstable_cache**: In-memory data cache with tags — busted via `revalidateTag()`
- **Supabase**: Source of truth, queried only on cache miss

## TTL Values

All TTLs are **24h (86400s)** — a safety net if the pipeline fails entirely.

| Location | TTL | Tag | On-demand |
|----------|-----|-----|-----------|
| `/news` page ISR | 86400 | — | `revalidatePath` |
| `/news` `unstable_cache` (×2) | 86400 | `news` | `revalidateTag` |
| `/news/[slug]` ISR | 86400 | — | `revalidatePath` |
| `sitemap.ts` ISR | 86400 | — | `revalidatePath` |
| `/rss.xml` `unstable_cache` | 86400 | `news` | `revalidateTag` |
| `/rss.xml` CDN header | 86400 | — | CDN expiry |
| `/api/news` CDN header | 86400 | — | CDN expiry |

## On-Demand Revalidation Flow

```
Pipeline completes
  → POST /api/revalidate (x-revalidation-secret header)
    → revalidateTag('news')        # busts unstable_cache (news page + RSS)
    → revalidatePath('/news')      # regenerates ISR page
    → revalidatePath('/news/[slug]', 'page')
    → revalidatePath('/sitemap.xml')
    → revalidatePath('/rss.xml')
  → Cache warming (curl /news, categories, sitemap, RSS)
```

## Env Vars Required

| Variable | Where | Purpose |
|----------|-------|---------|
| `REVALIDATION_SECRET` | Vercel + Mac Mini `.env.pipeline` | Auth for revalidation endpoint |
| `SITE_URL` | Mac Mini `.env.pipeline` | Base URL for cache warming (`https://chopaul.com`) |

## Scaling

If pipeline frequency increases (e.g., 3×/day), **no code changes needed**. Each run calls `/api/revalidate` → cache busted instantly. The 24h TTL only activates when all pipeline runs fail for an entire day.

## TTS File Resolution

Local TTS files (chapters, sentences, transcripts) are resolved dynamically by scanning `notebooks/tts_outputs/text/` for the latest `chapters-en-*.json` file. No hardcoded dates.

## Troubleshooting

- **Stale content after pipeline**: Check `REVALIDATION_SECRET` matches on Vercel and Mac Mini
- **Revalidation returns 401**: Secret mismatch — verify env vars
- **Revalidation returns non-200**: Check Vercel function logs
- **`x-vercel-cache: HIT` after revalidation**: Cache warming succeeded; if stale, check tag names match
- **TTS audio stale**: Verify `audio_url` in `wsj_briefings` table points to latest; fallback uses local file scan
