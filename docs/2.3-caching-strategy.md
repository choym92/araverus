# Caching Strategy

## Architecture

```
CDN (Vercel Edge) ‚Üí ISR (Next.js) ‚Üí unstable_cache ‚Üí Supabase
```

- **CDN**: `Cache-Control` headers for API routes (`/api/news`, `/rss.xml`)
- **ISR**: `export const revalidate` on page routes ‚Äî stale pages served while regenerating
- **unstable_cache**: In-memory data cache with tags ‚Äî busted via `revalidateTag()`
- **Supabase**: Source of truth, queried only on cache miss

## TTL Values

All TTLs are **24h (86400s)** ‚Äî a safety net if the pipeline fails entirely.

| Location | TTL | Tag | On-demand |
|----------|-----|-----|-----------|
| `/news` (all articles) ISR | 86400 | ‚Äî | `revalidateTag('news')` + tag busts all |
| `/news/c/[category]` (e.g., `/news/c/tech`) ISR | 86400 | ‚Äî | `revalidateTag('news')` + tag busts all |
| `/news` + `/news/c/*` `unstable_cache` | 86400 | `news` | `revalidateTag('news')` |
| `/news/[slug]` (article detail) ISR | 86400 | ‚Äî | `revalidatePath` (individual) |
| `sitemap.ts` ISR | 86400 | ‚Äî | `revalidatePath` |
| `/rss.xml` `unstable_cache` | 86400 | `news` | `revalidateTag` |
| `/rss.xml` CDN header | 86400 | ‚Äî | CDN expiry |
| `/api/news` CDN header | 86400 | ‚Äî | CDN expiry |
| `/sitemap-news.xml` `unstable_cache` | 3600 | `news` | `revalidateTag` |
| `/sitemap-news.xml` CDN header | 3600 | ‚Äî | CDN expiry |

## On-Demand Revalidation Flow

```
Pipeline completes
  ‚Üí POST /api/revalidate (x-revalidation-secret header)
    ‚Üí revalidateTag('news')        # busts unstable_cache (all /news + /news/c/* + RSS)
    ‚Üí revalidatePath('/news')      # regenerates ISR for /news
    ‚Üí revalidatePath('/news/c/[category]', 'page') ‚Äî REMOVED (no blanket pattern ‚Äî see Rule #2)
    ‚Üí revalidatePath('/news/[slug]', 'page')
    ‚Üí revalidatePath('/sitemap.xml')
    ‚Üí revalidatePath('/rss.xml')
  ‚Üí Cache warming (curl /news + all /news/c/[category] routes, sitemap, RSS)
```

**Important**: Category pages (`/news/c/tech`, etc.) are **not** individually revalidated via `revalidatePath()` ‚Äî only `revalidateTag('news')` is used. This prevents a cache storm where every category page regenerates at once. The `unstable_cache` tag busts all category pages' data at once, and individual pages refresh on-demand when visited after the tag bust.

## Scaling

If pipeline frequency increases (e.g., 3√ó/day), **no code changes needed**. Each run calls `/api/revalidate` ‚Üí cache busted instantly. The 24h TTL only activates when all pipeline runs fail for an entire day.

## TTS File Resolution

Local TTS files (chapters, sentences, transcripts) are resolved dynamically by scanning `notebooks/tts_outputs/text/` for the latest `chapters-en-*.json` file. No hardcoded dates.

## Critical Rules ‚Äî DO NOT BREAK

### 1. NEVER use `searchParams` in Server Components on cached pages

`searchParams` in a Server Component (page.tsx `generateMetadata` or default export) forces Next.js to treat the page as **fully dynamic**, bypassing ISR entirely ‚Äî even with `revalidate = 86400`. Every request becomes a MISS hitting the origin server.

**Lesson learned (2026-03-03):** Early `/news` design used `searchParams` for category filtering. Result: 130 req/min all MISS, consumed 7.5GB in 3 days. Fix: moved categories to **route segments** (`/news/c/[category]`) for independent ISR, and read remaining query params (tab, keywords) in `NewsContent` via `useSearchParams()`.

**Rule:**
- Use **route segments** for structural grouping that needs independent caching (categories, date ranges, etc.)
- Read cosmetic/filtering query params in client components with `useSearchParams()`, never in the server component

### 2. NEVER use `revalidatePath` with dynamic route patterns

`revalidatePath("/news/[slug]", "page")` invalidates ALL matching pages at once. With 100+ articles, this creates a cache storm where every subsequent visit triggers a fresh render.

**Applied to categories (2026-03-03):** `/news/c/[category]` routes do **not** use individual `revalidatePath()` calls. Instead, `revalidateTag('news')` busts the `unstable_cache` for all category pages' data at once. Each category page regenerates on the next visit without thundering herd.

**Rule:**
- Use `revalidateTag()` for bulk data invalidation across multiple related pages
- Use `revalidatePath('/specific/non-dynamic/path')` for individual static routes only
- NEVER use `revalidatePath('/route/[dynamic]', 'page')` without a concrete path

### 3. Visibility gate = headline required

The visibility gate in `getNewsItems()` hides ALL articles without an AI headline, regardless of age. Headline only exists on `relevance_flag='ok'` crawls (Step 2 LLM output). No headline = no display anywhere (list, detail, RSS, sitemap).

## Troubleshooting

- **Stale content after pipeline**: Check `REVALIDATION_SECRET` matches on Vercel and Mac Mini
- **Revalidation returns 401**: Secret mismatch ‚Äî verify env vars
- **Revalidation returns non-200**: Check Vercel function logs
- **`x-vercel-cache: HIT` after revalidation**: Cache warming succeeded; if stale, check tag names match
- **TTS audio stale**: Verify `audio_url` in `wsj_briefings` table points to latest; fallback uses local file scan
- **All requests MISS on Vercel logs**: Check if any Server Component uses `searchParams` ‚Äî see Rule #1 above
- **Cache storm (mass MISS after pipeline)**: Check `/api/revalidate` for blanket `revalidatePath` with dynamic patterns ‚Äî see Rule #2 above
