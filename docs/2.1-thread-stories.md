# Stories Tab ‚Äî Hierarchical Thread View

Stories tab (`/news?tab=stories`) displays story threads in a hierarchical accordion view: Parent Thread (LLM-generated group title) ‚Üí Sub-threads (existing `wsj_story_threads`) ‚Üí Articles. Currently all threads are flat (no parents yet ‚Äî backend pending). Category filtering is supported server-side.

For general `/news` architecture and data flow, see `docs/2-news-frontend.md`. For threading algorithm and heat scoring, see `docs/1.2-news-threading.md`.

```
page.tsx (Server Component)

```

## Heat Score Formula

Heat per sub-thread (calculated in service layer):

```
heat = Œ£ (importance_weight √ó e^(-0.3 √ó days_old))

where:
  importance_weight:
    must_read     = 3
    worth_reading = 2
    optional      = 1
  days_old = (now - article.created_at) / 86400 seconds
```

**Parent totalHeat**: Sum of all sub-thread heats in that group.

**Rationale**: Exponential decay (`e^(-0.3 √ó days_old)`) means articles lose ~26% heat per day, keeping recent threads at the top while older high-importance threads still rank above new low-importance ones.

## Category Filtering

**Server-side** via `?category=` query parameter.

**Logic in `getStoriesData()`**:
1. Filter `wsj_items` by `feed_name = category`
2. Return only threads that have ‚â•1 article in that category
3. For each thread, include only articles from the selected category in `recentArticles`

**Note**: Threads are cross-category (same thread may have articles from multiple feeds). Filtering by category shows only the relevant articles within each thread.

## Known Limitations

1. **Flat structure (temporary)**
   - All threads currently have `parent_id = null` until backend migration completes
   - UI still renders correctly ‚Äî all threads appear under a single "Ungrouped" section

2. **Heat calculation in JS** (acceptable for ‚â§50 threads)
   - Calculated in service layer, not in SQL
   - Trade-off: flexible formula vs. query complexity
   - If ‚â•100 threads become typical, move to SQL computed column or trigger

3. **Category filter at article level**
   - Filtering applies to articles within threads, not to threads themselves
   - A thread with 5 articles (3 TECH + 2 ECONOMY) will show only 3 articles when filtered by TECH
   - This is correct behavior ‚Äî threads are cross-category groupings

4. **No sub-thread drag/reorder**
   - Accordion is read-only
   - Sorting is by heat only; no user-customizable order
