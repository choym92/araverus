# Thread Stories — Frontend

Stories tab (`/news?tab=stories`) displays story threads in a hierarchical accordion view. For general `/news` architecture, see `docs/2-news-frontend.md`. For the threading algorithm and backend, see `docs/1.3-news-threading.md`.

---

## 1. Current Implementation

### Data Flow

```
page.tsx (Server Component)
  -> getStoriesData(category?) -- unstable_cache, 30-min TTL, tag: 'news'
    -> NewsService.getActiveThreadsGrouped(category?)
      -> Supabase: wsj_story_threads (active) + wsj_items (articles per thread)
      -> Calculate heat score per thread
      -> Group by parent_id (null = orphan group)
      -> Sort by totalHeat desc
  -> Pass parentThreadGroups to NewsContent
    -> NewsContent renders StoriesTab component
```

### Interfaces

```typescript
interface ParentThread {
  id: string;
  title: string;
}

interface StoryThreadWithDetails extends StoryThread {
  parent_id: string | null;
  heat: number;
  recentArticles: NewsItem[];
}

interface ParentThreadGroup {
  parent: ParentThread | null;  // null for orphan/ungrouped threads
  subThreads: StoryThreadWithDetails[];
  totalHeat: number;
}
```

### StoriesTab Component

**File**: `src/app/news/_components/StoriesTab.tsx` (Client Component)

**Props**: `{ groups: ParentThreadGroup[] }`

**Features**:
- Accordion UI — click sub-thread row to expand/collapse article list (single-select)
- Heat badge: dot indicator based on thresholds (3 dots >= 8, 2 dots >= 4, 1 dot >= 1)
- Importance pills: Must Read (black), Worth Reading (neutral)
- Empty state: "No active stories."

**Render tree**:
```
StoriesTab
  ParentThreadGroup (if parent exists, group header)
    SubThreadRow (click to expand)
      Thread title + heat badge + importance pill count
      [EXPANDED] Article list
        ArticleRow (title + source + date)
  [EMPTY] "No active stories."
```

### Other Thread UI Surfaces

1. **ArticleCard carousel** — card footer `< 3/7 > Thread Title` slide
2. **TimelineSection** — `/news/[slug]` article detail page vertical timeline

### Heat Score Formula

```
heat = SUM(importance_weight * e^(-0.3 * days_old))

importance_weight:
  must_read     = 3
  worth_reading = 2
  optional      = 1

days_old = (now - article.created_at) / 86400
```

Parent totalHeat = sum of all sub-thread heats. Exponential decay (~26%/day) keeps recent threads on top while high-importance older threads still rank above new low-importance ones.

### Category Filtering

Server-side via `?category=` query parameter. Threads are cross-category — filtering shows only matching articles within each thread. Threads with zero matching articles are hidden.

### Tab Navigation

```typescript
const isDisabled = t.value === 'search';  // only Search tab is disabled
```

URL patterns:
- `/news?tab=stories` — all categories
- `/news?tab=stories&category=TECH` — Tech only

## 3. Causal Thread Links (Future)

Current threading groups by **topical similarity** — articles about the same event/entity. It cannot link causally related but topically different threads:
```
"Fed Cuts Interest Rates" -> "S&P 500 Surges 3%" -> "Housing Market Heats Up"
```

These have low cosine similarity (~0.55-0.65) because they're different topics, even though they're causally connected.

### Concept

Keep topical threading as-is. Add a separate causal relationship layer on top:
```
Topical threads (existing):
  [Fed Rate Policy]   3 dots
  [Stock Market]      3 dots
  [Housing Market]    2 dots

Causal links (new layer):
  [Fed Rate Policy] --causes-> [Stock Market]
  [Fed Rate Policy] --causes-> [Housing Market]
  [Stock Market]    --causes-> [Housing Market]
```

### Schema

```sql
CREATE TABLE wsj_thread_links (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    source_thread_id UUID NOT NULL REFERENCES wsj_story_threads(id),
    target_thread_id UUID NOT NULL REFERENCES wsj_story_threads(id),
    relationship TEXT NOT NULL,        -- 'causes', 'reacts_to', 'escalates'
    confidence FLOAT,                  -- LLM confidence score
    evidence_summary TEXT,             -- LLM reasoning
    created_at TIMESTAMPTZ DEFAULT now(),
    UNIQUE(source_thread_id, target_thread_id)
);
```

### Detection Pipeline

After daily threading:
1. Collect active threads with titles + recent article titles
2. Send to Gemini: "Which threads have causal relationships?"
3. Store links in `wsj_thread_links`
4. Purely LLM judgment — no embedding validation needed

### Frontend: "Related Storylines"

On article detail page, below Story Timeline:
```

|  This story is connected to:                   |
|                                                |
```

### Many-to-Many (Optional)

If causal links exist, articles could belong to multiple threads:
```sql
CREATE TABLE wsj_item_threads (
    wsj_item_id UUID REFERENCES wsj_items(id),
    thread_id UUID REFERENCES wsj_story_threads(id),
    relationship TEXT DEFAULT 'member',  -- 'member' | 'related' | 'caused_by'
    PRIMARY KEY (wsj_item_id, thread_id)
);
```

Replaces `wsj_items.thread_id` FK. Defer until Stories tab is mature.

## 4. Known Limitations & Open Questions

### Current Limitations

1. **Flat structure** — All threads have `parent_id = null` until backend migration adds `wsj_parent_threads` table
2. **Heat calculation in JS** — Acceptable for <=50 threads; move to SQL if >=100 become typical
3. **Category filter at article level** — Correct behavior (threads are cross-category), but can be confusing
4. **No sub-thread reorder** — Accordion is read-only, sorted by heat only
5. **1:1 FK constraint** — `wsj_items.thread_id` means one article belongs to one thread
6. **Only title + description embedded** — Full article body with causal language isn't used

### Open Questions

- **Thread quality monitoring**: Add avg pairwise similarity, member count distribution, creation/archive rate to `pipeline_health.py`
- **Thread title quality**: Are Gemini-generated titles good enough for user-facing display?
- **Category-level thread exposure**: Currently global heat ranking — no guarantee of "top 3 per category" coverage. Frontend sorting change needed.
- **Centroid alternatives**: Mean (current) drifts with diverse members. Medoid (O(n^2)) or LLM-generated text embedding (API cost) are options, but not worth changing until monitoring shows problems.

### Backend Dependencies (Pending)

For parent threading hierarchy:

1. `wsj_parent_threads` table (`id`, `title`, `created_at`)
2. `wsj_story_threads.parent_id` FK column
3. Service layer update to select `parent_id` and fetch parent titles
