## 1. Daily Pipeline Flow

```mermaid
flowchart TD
    A["New articles crawled<br/>(RSS: title + description)"] --> C["Embed with BAAI/bge-base-en-v1.5<br/>(title + description + summary -> 768-dim vector)"]
    C --> E["Compare article vector against<br/>every Active + Cooling thread's<br/>time-weighted centroid"]

    E --> F{"Best thread similar enough?<br/>(adjusted_sim >= base 0.60<br/>+ time penalty)<br/><br/>Author boost: threshold drops<br/>to 0.55 if same creator<br/>within 48h"}
    F -->|Yes| G{"Clear winner?<br/>(best - 2nd best >= 0.03)<br/>e.g. 0.80 vs 0.71 = clear<br/>e.g. 0.80 vs 0.78 = ambiguous"}
    G -->|Yes| H["Assign to best-matching thread"]
    G -->|No| I["Ambiguous -> leave Unmatched"]
    F -->|No| I

    H --> J["Update stored centroid<br/>(running mean)"]

    I --> L["Gemini 2.5 Flash LLM grouping<br/>(title + date + author + summary + keywords)"]

    L --> M["LLM returns candidate groups<br/>(max 8 articles per group)"]
    M --> P{"Merge check:<br/>new centroid vs existing<br/>> 0.92?"}

    P -->|Yes| Q["Merge into existing thread"]
    P -->|No| R["Create new thread"]

    R --> CE["CE merge pass:<br/>find similar thread pairs<br/>(centroid >= 0.60)<br/>confirm with cross-encoder<br/>(GTE-ModernBERT >= 0.85)<br/>merge confirmed pairs"]
    Q --> CE
    H --> CE

    CE --> S["Heat scores auto-update<br/>(query-time calculation)"]

    style A fill:#4a9eff,color:#fff
    style H fill:#2ecc71,color:#fff
    style Q fill:#f39c12,color:#fff
    style R fill:#9b59b6,color:#fff
    style I fill:#e74c3c,color:#fff
    style CE fill:#e67e22,color:#fff
```

## 2. Thread Lifecycle

```mermaid
stateDiagram-v2
    [*] --> Active: New thread created<br/>(from LLM grouping)

    Active --> Active: New article matched<br/>(resets last_seen)
    Active --> Cooling: No new articles<br/>for 3+ days
    Cooling --> Active: New article matched<br/>(resets last_seen)
    Cooling --> Archived: 14+ days inactive<br/>(status='archived')
    Archived --> Active: Resurrected!<br/>Cooling thread matched<br/>(status='active')

    note right of Active
        DB: status='active'
        last_seen < 3 days ago
        Matching: base threshold (0.60)
        Frontend: shown, ranked by heat
    end note

    note right of Cooling
        DB: status='cooling'
        last_seen 3-14 days ago
        Matching: higher threshold (time penalty)
        Frontend: shown below active
    end note

    note right of Archived
        DB: status='archived'
        last_seen 14+ days ago
        Matching: NOT in matching pool
        Frontend: hidden from feed,
        accessible via article detail
    end note
```

## 3. Dynamic Threshold

```mermaid
flowchart TD
    START["New article arrives"] --> CALC["Calculate effective_threshold<br/>for each Active + Cooling thread"]

    CALC --> BASE["base = 0.60"]
    CALC --> TIME["time_penalty = 0.005 x days_gap<br/>(days since thread's last article)"]
    CALC --> ENTITY["entity_boost = ENTITY_WEIGHT x<br/>IDF-weighted Jaccard overlap<br/>(added to similarity, not threshold)"]

    BASE --> SUM["effective_threshold =<br/>base + time_penalty"]
    TIME --> SUM

    SUM --> AUTHOR{"Same creator wrote in<br/>this thread within 48h?"}
    AUTHOR -->|Yes| BOOST["threshold = 0.55<br/>(author boost)"]
    AUTHOR -->|No| CAP

    BOOST --> CAP{"member_count >= 50?"}
    CAP -->|Yes| HARDCAP["threshold =<br/>max(computed, 0.87)<br/>Hard cap: near-frozen"]
    CAP -->|No| FINAL["Use computed threshold"]
    HARDCAP --> COMPARE
    FINAL --> COMPARE

    COMPARE{"adjusted_sim = cosine_sim<br/>+ entity_boost<br/>>= threshold?"}
    COMPARE -->|Yes| MARGIN{"Margin check:<br/>best - 2nd best >= 0.03?"}
    COMPARE -->|No| UNMATCHED["Unmatched"]
    MARGIN -->|Yes| MATCHED["Matched"]
    MARGIN -->|No| AMBIG["Ambiguous -> Unmatched"]

    subgraph "Threshold Examples"
        EX1["0 days gap -> 0.600"]
        EX2["3 days gap -> 0.615"]
        EX3["7 days gap -> 0.635"]
        EX4["14 days gap -> 0.670"]
    end

    style MATCHED fill:#2ecc71,color:#fff
    style UNMATCHED fill:#e74c3c,color:#fff
    style AMBIG fill:#e67e22,color:#fff
    style HARDCAP fill:#3498db,color:#fff
    style BOOST fill:#9b59b6,color:#fff
```

## 4. Centroid: Storage vs Matching

```mermaid
flowchart TD
    subgraph STORAGE ["Stored Centroid (Running Mean)"]
        S1["When article joins thread:"]
        S2["new_centroid = normalize(<br/>mean of ALL member embeddings)"]
        S3["Simple average, unweighted"]
        S4["Used as fallback only"]
        S1 --> S2 --> S3 --> S4
    end

    subgraph MATCHING ["Matching Centroid (Time-Weighted)"]
        M1["At match time, recompute:"]
        M2["weight_i = exp(-0.10 x days_old)"]
        M3["centroid = normalize(<br/>sum(weight_i x embedding_i))"]
        M4["Recent articles dominate"]
        M1 --> M2 --> M3 --> M4
    end

    subgraph DECAY ["Time Decay (CENTROID_DECAY = 0.10)"]
        D1["Today: weight 1.000 (100%)"]
        D2["3 days ago: weight 0.741 (74%)"]
        D3["7 days ago: weight 0.497 (50%)"]
        D4["14 days ago: weight 0.247 (25%)"]
    end

    subgraph WHY ["Why Time-Weighted Replaces EMA"]
        W1["EMA was insertion-order based<br/>(alpha depended on member_count at join time)"]
        W2["Early articles had permanent<br/>outsized influence"]
        W3["Time-weighted recalculates fresh:<br/>old articles lose influence over time<br/>regardless of when they joined"]
    end

    style M4 fill:#2ecc71,color:#fff
    style S4 fill:#95a5a6,color:#fff
```

## 5. LLM Grouping

```mermaid
flowchart TD
    IN["Unmatched articles<br/>(accumulated over 3-day window)"] --> LLM["Send to Gemini 2.5 Flash<br/>Per article: title + date + author<br/>+ keywords + summary (2 sentences)"]

    LLM --> GROUPS["LLM returns groups<br/>with suggested thread titles + summaries"]

    GROUPS --> SIZECAP{"Group > 8 articles?"}
    SIZECAP -->|Yes| TRUNCATE["Truncate to 8"]
    SIZECAP -->|No| MERGE_CHECK
    TRUNCATE --> MERGE_CHECK

    MERGE_CHECK{"For each group:<br/>cos_sim(new_centroid,<br/>existing_centroid) > 0.92?"}
    MERGE_CHECK -->|Yes| MERGE["Merge into<br/>existing thread"]
    MERGE_CHECK -->|No| CREATE["Create new thread<br/>with LLM title + summary"]

    subgraph "Why This Exists"
        WHY1["Centroid matching only assigns<br/>to EXISTING threads"]
        WHY2["New topics need LLM to create<br/>the initial thread"]
        WHY3["Without it, new stories<br/>would never get threaded"]
    end

    subgraph "LLM Prompt Rules"
        R1["Specific events only<br/>(no generic topics)"]
        R2["5-10 word titles<br/>naming entities/actions"]
        R3["Author as grouping hint"]
        R4["Max 8 articles per group"]
        R5["No cross-validation<br/>(removed: cosine can't<br/>capture causal links)"]
    end

    style MERGE fill:#f39c12,color:#fff
    style CREATE fill:#9b59b6,color:#fff
```

## 7. Heat Score

```mermaid
flowchart TD
    QUERY["Frontend queries threads"] --> CALC["For each thread,<br/>sum over all member articles:"]
    CALC --> FORMULA["heat = sum(importance_weight x time_decay)"]

    FORMULA --> WEIGHT["importance_weight:<br/>must_read = 3<br/>worth_reading = 2<br/>optional = 1"]
    FORMULA --> DECAY["time_decay:<br/>e^(-0.3 x days_old)<br/>half-life ~ 2.3 days"]

    WEIGHT --> EXAMPLES
    DECAY --> EXAMPLES

    subgraph EXAMPLES ["Heat Score Examples"]
        EX1["'Fed Rate Decision'<br/>today: 2x must_read<br/>yesterday: 3x worth_reading<br/>-> heat ~ 10.5"]
        EX2["'Venezuela Crisis'<br/>last article 5 days ago<br/>1x must_read<br/>-> heat ~ 0.67"]
    end

    EXAMPLES --> RANK["Threads ranked by heat<br/>in frontend feed"]

    style EX1 fill:#e74c3c,color:#fff
    style EX2 fill:#3498db,color:#fff
```

## 8. Edge Cases

```mermaid
flowchart TD
    subgraph QUALITY ["Thread Quality -- Preventing Contamination"]
        direction LR
        EC1["Gravity Well<br/>(large thread absorbs all)"]
        EC1 --> S1["Time-weighted centroid:<br/>large old thread's centroid<br/>reflects recent articles only.<br/>Tangential article scores low.<br/>Hard cap at 50 members (0.87)"]

        EC2["Centroid Drift<br/>(thread -> general news)"]
        EC2 --> S2["Running mean + time-weighted:<br/>off-topic articles diluted by<br/>existing members + time decay.<br/>Centroid barely shifts."]

        EC3["Roundup Pollution<br/>(Market Talk matches all)"]
        EC3 --> S3["Filtered at RSS ingest<br/>(1_wsj_ingest.py)<br/>Never enters pipeline"]

        EC4["LLM Garbage Groups<br/>(China EV + China Earthquake)"]
        EC4 --> S4["Strict prompt rules +<br/>summary context helps LLM<br/>distinguish same-keyword<br/>different-event articles"]
    end

    subgraph LIFECYCLE ["Thread Lifecycle"]
        direction LR
        EC5["Near-Duplicate Threads"]
        EC5 --> S5["Merge check > 0.92<br/>+ CE merge pass (0.85)"]

        EC6["Same Topic, Diff Events<br/>(two separate shootings)"]
        EC6 --> S6["Time penalty separates:<br/>old thread needs higher sim.<br/>CE merge won't confirm<br/>(different representative articles)"]

        EC7["Story Evolution<br/>(trade war splits)"]
        EC7 --> S7["Sub-stories naturally split:<br/>diverging embeddings fail<br/>to match old centroid.<br/>LLM creates focused threads."]

        EC8["Resurrection<br/>(quiet story re-erupts)"]
        EC8 --> S8["Cooling threads still in<br/>matching pool (higher threshold).<br/>Match -> status='active'.<br/>Archived -> NOT in pool,<br/>LLM creates new thread."]
    end

    subgraph VOLUME ["Volume & Assignment"]
        direction LR
        EC9["Volume Spike<br/>(50 articles/day)"]
        EC9 --> S9["Linear scaling.<br/>First articles create threads,<br/>later ones match them."]

        EC10["Article Fits 2 Threads"]
        EC10 --> S10["Margin < 0.03 -> ambiguous<br/>-> unmatched -> LLM decides.<br/>1:1 FK constraint."]

        EC11["New Topic Appears"]
        EC11 --> S11["Fails centroid matching.<br/>LLM grouping creates thread<br/>if 2+ articles found."]
    end
```

## 10. Algorithm Constants

```mermaid
mindmap
  root((Threading<br/>Constants))
    Matching
      Base threshold: 0.60
      Time penalty: 0.005/day
      Author boost: 0.55
      Author window: 48h
      Hard cap: 50 members -> 0.87
      Match margin: 0.03
      Entity weight: 0.04 (IDF Jaccard)
    Centroid
      Storage: running mean (unweighted)
      Matching: time-weighted (decay=0.10)
    LLM Grouping
      Max group size: 8
      Window: 3 days
      Input: title + date + author + summary + keywords
    Merge
      LLM merge threshold: 0.92
    CE Merge Pass
      Model: GTE-ModernBERT (149M)
      Centroid prefilter: 0.60
      CE threshold: 0.85
    Lifecycle
      Cooling: 3+ days
      Archive: 14+ days
    Heat Score
      must_read weight: 3
      worth_reading weight: 2
      optional weight: 1
      Decay: e^(-0.3 x days)
```
