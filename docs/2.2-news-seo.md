# 2.2 News SEO

## Overview

SEO configuration for araverus.com ŌĆö metadata, structured data, sitemap, RSS, and robots strategy. Primary focus: maximizing Google discoverability of `/news` content. Blog is intentionally de-prioritized (hidden from sitemap and RSS).

---

## Site-wide (`src/app/layout.tsx`)

### Metadata
- **title**: `{ default: "Paul Cho ŌĆö AI Projects on News and Finance", template: "%s | Paul Cho" }`
- **description**: `"Using agentic AI and neural networks to get insights on news and finance."`
- **metadataBase**: `https://araverus.com` ŌĆö resolves relative URLs in openGraph/twitter
- **openGraph**: type `website`, locale `en_US`, siteName `araverus.com`, image `og-news-default.png` (1200x630)
- **twitter**: `summary_large_image` card, same image
- **alternates**: RSS link at `/rss.xml`

### JSON-LD (root `<body>`)
Two schemas injected on every page:

```json
[
  {
    "@type": "WebSite",
    "name": "araverus.com",
    "url": "https://araverus.com",
    "description": "Using agentic AI and neural networks to get insights on news and finance."
  },
  {
    "@type": "Person",
    "name": "Paul Cho",
    "url": "https://araverus.com",
    "sameAs": ["https://www.linkedin.com/in/ympcho/"]
  }
]
```

**Purpose**: Google Knowledge Panel eligibility. Links Paul Cho identity to araverus.com with LinkedIn verification via `sameAs`.

### Fonts
All three fonts use `display: 'swap'` to prevent invisible text during load (CLS optimization):
- Inter (body)
- Geist Mono (code)
- Playfair Display (serif headings)

## `/news` page (`src/app/news/page.tsx`)

### Metadata
- **title**: `{ absolute: titleText }` ŌĆö bypasses `| Paul Cho` template
  - Default: `"AI News Briefing ŌĆö Tech, Markets & Finance"`
  - Per category: `"${label} News ŌĆö AI Briefing"` (e.g. `"Tech News ŌĆö AI Briefing"`)
- **description**: threading-focused, dynamic per category
  - Default: `"Agentic AI pipeline that threads related news stories, surfaces trends, and delivers daily briefings across Tech, Markets, and Finance."`
- **canonical**: absolute URL ŌĆö `https://araverus.com/news` or `https://araverus.com/news?category=${category}`
- **openGraph**: title as string (not object), type `website`, image `og-news-default.png` (1200x630), **og:audio** (latest EN briefing `audio_url` when available)
- **twitter**: `summary_large_image`

### JSON-LD ŌĆö PodcastEpisode
Rendered as `<script type="application/ld+json">` in page component (not metadata). Links to PodcastSeries via `partOfSeries`. Includes `associatedMedia` with audio URL, encoding format, and ISO 8601 duration.

### JSON-LD ŌĆö PodcastSeries (layout)
Injected in `news/layout.tsx` ŌĆö applies to all `/news/*` pages. Links show to RSS feed at `/podcast.xml`.

### Metadata

- **canonical**: `https://araverus.com/news/${slug}` (absolute)
- **openGraph**: type `article`, image uses `item.top_image` (no dimensions declared for external images) or `og-news-default.png` fallback (1200x630 declared)
- **twitter**: `summary_large_image`, always

### JSON-LD ŌĆö NewsArticle + BreadcrumbList

Two schemas per article page:

**NewsArticle:**

| Field | Source | Notes |
|-------|--------|-------|
| `url` | `https://araverus.com/news/${slug}` | Absolute URL |
| `headline` | `wsj_items.title` | |
| `description` | `wsj_items.description` | RSS original |
| `articleBody` | `wsj_llm_analysis.summary` fallback `description` | AI-generated summary |
| `articleSection` | `feed_name` ŌåÆ label (TECH ŌåÆ Tech) | |
| `inLanguage` | `"en"` | |
| `isAccessibleForFree` | `true` (boolean, not string) | No paywall |
| `datePublished` | `wsj_items.published_at` | ISO 8601 |
| `dateModified` | `wsj_items.published_at` | Same as published (articles not edited) |
| `author` | `{ "@type": "Person", "name": "Paul Cho" }` | Paul Cho as curator/author |

**BreadcrumbList:**
```
News (position 1) ŌåÆ Category (position 2) ŌåÆ Article Title (position 3)
```
All items include `item` (URL) field. Last item uses `articleUrl`.

## Sitemap (`src/app/sitemap.ts`)

| URL | Priority | changeFrequency | lastModified |
|-----|----------|-----------------|-------------|
| `/` | 1.0 | weekly | now |
| `/news` | 1.0 | daily | latest article `published_at` |
| `/podcast.xml` | 0.6 | daily | latest article `published_at` |
| `/news/c/*` (x5) | 0.9 | daily | latest article `published_at` |
| `/news/[slug]` (latest 100) | 0.8 | weekly | article `published_at` |

- Blog routes intentionally excluded (blog de-prioritized)
- `revalidate = 3600` ŌĆö sitemap refreshes every hour
- Fetch failure caught silently ŌĆö sitemap still serves static routes
- `getNewsItems` returns descending by `published_at` ŌĆö `[0]` is always the latest

### Why no Google News Sitemap?
araverus.com is not a registered Google News Publisher. `<news:publication>` format only benefits approved sites.

## Podcast Feed (`/podcast.xml`)

Podcast RSS 2.0 feed with iTunes and Atom namespaces. Exposes the latest EN audio briefing for podcast distribution (Apple Podcasts, Spotify, Google Podcasts).

### Channel
- **title**: `"AI News Briefing ŌĆö araverus.com"`
- **itunes:author**: `araverus.com`
- **itunes:category**: `News > Daily News`
- **itunes:image**: `podcast-cover.png` (1400x1400 square)
- **itunes:explicit**: `false`
- **itunes:type**: `episodic`
- **atom:link**: self-reference at `/podcast.xml`

### Items
- Single `<item>` with latest EN briefing from `NewsService.getLatestBriefings()`
- `<enclosure>` points to `audio_url` from `wsj_briefings` table
- `<guid>`: `briefing-en-YYYY-MM-DD` (not permalink)
- `<itunes:duration>`: from `audio_duration` column (seconds)

### Cache
- `unstable_cache` with `['news']` tag, 24h revalidation
- `s-maxage=86400, stale-while-revalidate`

### Constraint
Only latest episode exposed (Supabase free tier 1GB). Same URL, content refreshes daily.

## RSS Feed (`/rss.xml`)

News-only feed (blog removed).

### Channel
- **title**: `"araverus.com ŌĆö AI-Powered News & Finance Insights"`
- **description**: threading pipeline description
- **language**: `en`
- **copyright**: dynamic year
- **ttl**: `60` (minutes)
- **lastBuildDate**: latest article `published_at` (not current time)
- **image**: `logo-publisher.png` (no width/height declared)
- **atom:link**: self-reference for feed validators

### Items
- Latest 50 news articles from `NewsService.getNewsItems()`
- Each item: title, link, guid, pubDate, description, category (feed_name)

### Cache
- `s-maxage=3600, stale-while-revalidate` (1 hour)
- SITE fallback: `https://araverus.com` (not localhost)

## robots.ts

**Rule 1 ŌĆö All crawlers (`*`)**
```
Allow: /
Disallow: /dashboard, /admin
```

**Rule 2 ŌĆö Googlebot-News**
```
Allow: /news/
```
Path prefix covers `/news`, `/news/[slug]`, `/news?category=*`. Googlebot-News is separate from regular Googlebot ŌĆö indexes for Google News surfaces (News tab, Discover, Top Stories).

Note: Requires Google News Publisher approval to appear in Google News. This rule removes technical barriers.

## Search Console Checklist

After deploying:
1. Register `araverus.com` at [search.google.com/search-console](https://search.google.com/search-console)
2. Submit `https://araverus.com/sitemap.xml`
3. Submit `https://araverus.com/rss.xml`
4. URL Inspection on `/news` ŌåÆ request indexing
5. Verify JSON-LD at [Rich Results Test](https://search.google.com/test/rich-results)
6. Check Core Web Vitals report (font `display: swap` should help CLS)
