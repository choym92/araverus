# 2.2 News SEO

## Overview

SEO configuration for chopaul.com ‚Äî metadata, structured data, sitemap, RSS, and robots strategy. Primary focus: maximizing Google discoverability of `/news` content. Blog is intentionally de-prioritized (hidden from sitemap and RSS).

---

## Site-wide (`src/app/layout.tsx`)

### Metadata
- **title**: `{ default: "Paul Cho ‚Äî AI Projects on News and Finance", template: "%s | Paul Cho" }`
- **description**: `"Using agentic AI and neural networks to get insights on news and finance."`
- **metadataBase**: `https://chopaul.com` ‚Äî resolves relative URLs in openGraph/twitter
- **openGraph**: type `website`, locale `en_US`, siteName `chopaul.com`, image `og-news-default.png` (1200x630)
- **twitter**: `summary_large_image` card, same image
- **alternates**: RSS link at `/rss.xml`

### JSON-LD (root `<body>`)
Two schemas injected on every page:

```json
[
  {
    "@type": "WebSite",
    "name": "chopaul.com",
    "url": "https://chopaul.com",
    "description": "Using agentic AI and neural networks to get insights on news and finance."
  },
  {
    "@type": "Person",
    "name": "Paul Cho",
    "url": "https://chopaul.com",
    "sameAs": ["https://www.linkedin.com/in/ympcho/"]
  }
]
```

**Purpose**: Google Knowledge Panel eligibility. Links Paul Cho identity to chopaul.com with LinkedIn verification via `sameAs`.

### Fonts
All three fonts use `display: 'swap'` to prevent invisible text during load (CLS optimization):
- Inter (body)
- Geist Mono (code)
- Playfair Display (serif headings)

## `/news` page (`src/app/news/page.tsx`)

### Metadata
- **title**: `{ absolute: titleText }` ‚Äî bypasses `| Paul Cho` template
  - Default: `"AI News Briefing ‚Äî Tech, Markets & Finance"`
  - Per category: `"${label} News ‚Äî AI Briefing"` (e.g. `"Tech News ‚Äî AI Briefing"`)
- **description**: threading-focused, dynamic per category
  - Default: `"Agentic AI pipeline that threads related news stories, surfaces trends, and delivers daily briefings across Tech, Markets, and Finance."`
- **canonical**: absolute URL ‚Äî `https://chopaul.com/news` or `https://chopaul.com/news?category=${category}`
- **openGraph**: title as string (not object), type `website`, image `og-news-default.png` (1200x630)
- **twitter**: `summary_large_image`

## `/news/[slug]` page (`src/app/news/[slug]/page.tsx`)

### Metadata
- **title**: `item.title` (template appends `| Paul Cho`)
- **description**: `item.summary` fallback `item.description`
- **canonical**: `https://chopaul.com/news/${slug}` (absolute)
- **openGraph**: type `article`, image uses `item.top_image` (no dimensions declared for external images) or `og-news-default.png` fallback (1200x630 declared)
- **twitter**: `summary_large_image`, always

### JSON-LD ‚Äî NewsArticle + BreadcrumbList

Two schemas per article page:

**NewsArticle:**

| Field | Source | Notes |
|-------|--------|-------|
| `url` | `https://chopaul.com/news/${slug}` | Absolute URL |
| `headline` | `wsj_items.title` | |
| `description` | `wsj_items.description` | RSS original |
| `articleBody` | `wsj_llm_analysis.summary` fallback `description` | AI-generated summary |
| `articleSection` | `feed_name` ‚Üí label (TECH ‚Üí Tech) | |
| `inLanguage` | `"en"` | |
| `isAccessibleForFree` | `true` (boolean, not string) | No paywall |
| `datePublished` | `wsj_items.published_at` | ISO 8601 |
| `dateModified` | `wsj_items.published_at` | Same as published (articles not edited) |
| `author` | `{ "@type": "Person", "name": "Paul Cho" }` | Paul Cho as curator/author |
| `publisher.logo` | `logo-publisher.png` (600x60) | Separate file from main logo |
| `image` | `item.top_image` (no dimensions) or `og-news-default.png` (1200x630) | External images: no dimensions declared (unknown size) |
| `keywords` | `wsj_llm_analysis.keywords` joined by `, ` | LLM-extracted, 2-4 per article |

**BreadcrumbList:**
```
News (position 1) ‚Üí Category (position 2) ‚Üí Article Title (position 3)
```
All items include `item` (URL) field. Last item uses `articleUrl`.

### ISR
- `generateStaticParams`: pre-renders latest **100** articles at build
- `revalidate = 7200` (2 hours) ‚Äî content rarely changes after publish

## Sitemap (`src/app/sitemap.ts`)

| URL | Priority | changeFrequency | lastModified |
|-----|----------|-----------------|-------------|
| `/` | 1.0 | weekly | now |
| `/news` | 1.0 | daily | latest article `published_at` |
| `/news?category=*` (x5) | 0.9 | daily | latest article `published_at` |
| `/news/[slug]` (latest 100) | 0.8 | weekly | article `published_at` |

- Blog routes intentionally excluded (blog de-prioritized)
- `revalidate = 3600` ‚Äî sitemap refreshes every hour
- Fetch failure caught silently ‚Äî sitemap still serves static routes
- `getNewsItems` returns descending by `published_at` ‚Äî `[0]` is always the latest

### Why no Google News Sitemap?
chopaul.com is not a registered Google News Publisher. `<news:publication>` format only benefits approved sites.

## RSS Feed (`/rss.xml`)

News-only feed (blog removed).

### Channel
- **title**: `"chopaul.com ‚Äî AI-Powered News & Finance Insights"`
- **description**: threading pipeline description
- **language**: `en`
- **copyright**: dynamic year
- **ttl**: `60` (minutes)
- **lastBuildDate**: latest article `published_at` (not current time)
- **image**: `logo-publisher.png` (no width/height declared)
- **atom:link**: self-reference for feed validators

### Items
- Latest 50 news articles from `NewsService.getNewsItems()`
- Each item: title, link, guid, pubDate, description, category (feed_name)

### Cache
- `s-maxage=3600, stale-while-revalidate` (1 hour)
- SITE fallback: `https://chopaul.com` (not localhost)

## robots.ts

**Rule 1 ‚Äî All crawlers (`*`)**
```
Allow: /
Disallow: /dashboard, /admin
```

**Rule 2 ‚Äî Googlebot-News**
```
Allow: /news/
```
Path prefix covers `/news`, `/news/[slug]`, `/news?category=*`. Googlebot-News is separate from regular Googlebot ‚Äî indexes for Google News surfaces (News tab, Discover, Top Stories).

Note: Requires Google News Publisher approval to appear in Google News. This rule removes technical barriers.

## Search Console Checklist

After deploying:
1. Register `chopaul.com` at [search.google.com/search-console](https://search.google.com/search-console)
2. Submit `https://chopaul.com/sitemap.xml`
3. Submit `https://chopaul.com/rss.xml`
4. URL Inspection on `/news` ‚Üí request indexing
5. Verify JSON-LD at [Rich Results Test](https://search.google.com/test/rich-results)
6. Check Core Web Vitals report (font `display: swap` should help CLS)
