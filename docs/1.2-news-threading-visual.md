## 1. Daily Pipeline Flow

```mermaid
flowchart TD
    A["🔄 New articles crawled<br/>(RSS: title + description)"] --> C["Embed with BAAI/bge-base-en-v1.5<br/>(title + description + summary → 768-dim vector)"]
    C --> E["Compare article vector against<br/>every Active + Cooling thread's centroid<br/>(centroid = average vector of all members)<br/><br/>→ See Section 3 for full threshold calculation"]

    E --> F{"Is the best-matching thread<br/>similar enough?<br/>(cosine_sim ≥ base 0.73<br/>+ time penalty + size penalty)<br/><br/>Each thread has its own threshold —<br/>bigger/older threads need higher similarity"}
    F -->|Yes| G{"Is it a clear winner?<br/>(best thread score - 2nd best ≥ 0.03)<br/>e.g. 0.80 vs 0.71 = clear ✓<br/>e.g. 0.80 vs 0.78 = too close ✗"}
    G -->|Yes| H["✅ Assign to best-matching thread"]
    G -->|No| I["❌ Ambiguous — could belong to<br/>either thread → leave Unmatched"]
    F -->|No| I

    H --> J["Update centroid<br/>(Dynamic EMA)"]

    I --> K{"Article age ≤ 2 days?"}
    K -->|No| K_skip["Skip LLM grouping<br/>(too old)"]
    K -->|Yes| L["Gemini 2.5 Flash LLM grouping<br/>(batch unmatched articles)"]

    L --> M["LLM returns candidate groups<br/>(max 8 articles per group)"]
    M --> N{"Cross-validation:<br/>avg pairwise ≥ 0.60?<br/>min pair ≥ 0.25?"}
    N -->|Fail| O["❌ Reject group<br/>(outlier or garbage)"]
    N -->|Pass| P{"Merge check:<br/>new centroid vs existing<br/>> 0.92?"}

    P -->|Yes| Q["🔀 Merge into existing thread"]
    P -->|No| R["🆕 Create new thread"]

    R --> S["Heat scores auto-update<br/>(query-time calculation)"]
    Q --> S
    H --> S

    style A fill:#4a9eff,color:#fff
    style H fill:#2ecc71,color:#fff
    style Q fill:#f39c12,color:#fff
    style R fill:#9b59b6,color:#fff
    style O fill:#e74c3c,color:#fff
    style I fill:#e74c3c,color:#fff
    style K_skip fill:#95a5a6,color:#fff
```

## 2. Thread Lifecycle

```mermaid
stateDiagram-v2
    [*] --> Active: New thread created<br/>(from LLM grouping)

    Active --> Active: New article matched<br/>(resets last_seen)
    Active --> Cooling: No new articles<br/>for 3+ days
    Cooling --> Active: New article matched<br/>(resets last_seen)
    Cooling --> Archived: 14+ days inactive<br/>(active=false)
    Archived --> Active: Resurrected!<br/>New article matches<br/>(active=true)

    note right of Active
        DB: active=true
        last_seen < 3 days ago
        Matching: base threshold
        Frontend: shown, ranked by heat
    end note

    note right of Cooling
        DB: active=true
        last_seen 3-14 days ago
        Matching: higher threshold (time penalty)
        Frontend: shown below active
    end note

    note right of Archived
        DB: active=false
        last_seen 14+ days ago
        Matching: NO matching
        Frontend: hidden from feed,
        accessible via article detail
    end note
```

## 3. Dynamic Threshold (Anti-Gravity)

```mermaid
flowchart TD
    START["New article arrives"] --> CALC["Calculate effective_threshold"]

    CALC --> BASE["base = 0.73<br/>(tuned via reranker experiment)"]
    CALC --> TIME["time_penalty = 0.01 × days_gap<br/>(days since thread's last article)"]
    CALC --> SIZE["size_penalty = 0.04 × ln(member_count + 1)"]

    BASE --> SUM["effective_threshold =<br/>base + time_penalty + size_penalty"]
    TIME --> SUM
    SIZE --> SUM

    SUM --> CAP{"member_count ≥ 50?"}
    CAP -->|Yes| HARDCAP["effective_threshold =<br/>max(computed, 0.87)<br/>🧊 Hard cap: near-frozen"]
    CAP -->|No| FINAL["Use computed threshold"]
    HARDCAP --> COMPARE
    FINAL --> COMPARE

    COMPARE{"cosine_sim(article, centroid)<br/>≥ effective_threshold?"}
    COMPARE -->|Yes| MARGIN{"Margin check:<br/>best_sim - 2nd_best_sim ≥ 0.03?"}
    COMPARE -->|No| UNMATCHED["❌ Unmatched"]
    MARGIN -->|Yes| MATCHED["✅ Matched"]
    MARGIN -->|No| AMBIG["❌ Ambiguous → Unmatched<br/>(coin flip between threads)"]

    subgraph "Size Penalty Examples"
        EX1["2 members → +0.044 → 0.774"]
        EX2["10 members → +0.096 → 0.826"]
        EX3["50 members → +0.157 → 0.870 (hard cap)"]
        EX4["100 members → +0.185 → 0.870 (hard cap)"]
    end

    style MATCHED fill:#2ecc71,color:#fff
    style UNMATCHED fill:#e74c3c,color:#fff
    style AMBIG fill:#e67e22,color:#fff
    style HARDCAP fill:#3498db,color:#fff
```

```mermaid

    UPDATE --> EFFECT["Effect: new article's<br/>influence on centroid"]

    subgraph "Alpha by Thread Size"

        A4["150 members → alpha=0.020<br/>New article = 2% influence<br/>🪨 Centroid barely moves"]
    end

    EFFECT --> WHY["Why: Large threads stay stable<br/>Small threads remain flexible<br/>Off-topic article can't drag centroid"]
```

## 5. LLM Grouping + Cross-Validation

```mermaid
flowchart TD
    IN["Unmatched articles<br/>(from last 2 days only)"] --> FILTER["Filter: exclude articles<br/>older than 2 days"]
    FILTER --> LLM["Send to Gemini 2.5 Flash:<br/>'Which articles cover the<br/>same specific event?'<br/>(includes article dates)"]

    LLM --> GROUPS["LLM returns groups<br/>with suggested thread titles"]

    GROUPS --> SIZECAP{"Group > 8 articles?"}
    SIZECAP -->|Yes| TRUNCATE["Truncate to 8"]
    SIZECAP -->|No| VALIDATE
    TRUNCATE --> VALIDATE

    VALIDATE["validate_llm_group()"]
    VALIDATE --> AVG{"Avg pairwise<br/>cosine sim ≥ 0.60?"}
    AVG -->|No| REJECT["❌ Reject group<br/>(37% rejection rate)"]
    AVG -->|Yes| FLOOR{"Any pair<br/>cosine sim < 0.25?"}
    FLOOR -->|Yes| REJECT2["❌ Reject group<br/>(outlier detected)"]
    FLOOR -->|No| MERGE_CHECK

    MERGE_CHECK{"For each valid group:<br/>cos_sim(new_centroid,<br/>existing_centroid) > 0.92?"}
    MERGE_CHECK -->|Yes| MERGE["🔀 Merge into<br/>existing thread<br/>(avoid duplicates)"]
    MERGE_CHECK -->|No| CREATE["🆕 Create new thread<br/>with LLM-generated title"]

    subgraph "Why This Exists"
        WHY1["Centroid matching can only assign<br/>to EXISTING threads"]
        WHY2["New topics need LLM to create<br/>the initial thread"]
        WHY3["Without it, new stories<br/>would never get threaded"]
    end

    style REJECT fill:#e74c3c,color:#fff
    style REJECT2 fill:#e74c3c,color:#fff
    style MERGE fill:#f39c12,color:#fff
    style CREATE fill:#9b59b6,color:#fff
```

## 6. Heat Score

```mermaid
flowchart TD
    QUERY["Frontend queries threads"] --> CALC["For each thread,<br/>sum over all member articles:"]
    CALC --> FORMULA["heat = Σ (importance_weight × time_decay)"]

    FORMULA --> WEIGHT["importance_weight:<br/>must_read = 3<br/>worth_reading = 2<br/>optional = 1"]
    FORMULA --> DECAY["time_decay:<br/>e^(-0.3 × days_old)<br/>half-life ≈ 2.3 days"]

    WEIGHT --> EXAMPLES
    DECAY --> EXAMPLES

    subgraph EXAMPLES ["Heat Score Examples"]
        EX1["'Fed Rate Decision'<br/>today: 2× must_read<br/>yesterday: 3× worth_reading<br/>→ heat ≈ 10.5 🔥"]
        EX2["'Venezuela Crisis'<br/>last article 5 days ago<br/>1× must_read<br/>→ heat ≈ 0.67 🧊"]
    end

    EXAMPLES --> RANK["Threads ranked by heat<br/>in frontend feed"]

    style EX1 fill:#e74c3c,color:#fff
    style EX2 fill:#3498db,color:#fff
```

## 7. Edge Cases

```mermaid
flowchart TD
    subgraph QUALITY ["Thread Quality — Preventing Contamination"]
        direction LR
        EC1["🌀 Gravity Well<br/>(large thread absorbs all)"]
        EC1 --> S1["Size penalty 0.04×ln(n+1)<br/>+ hard cap at 50 members (0.87)<br/><br/>e.g. 'U.S. Economy' at 40 members<br/>threshold = 0.879, blocks 0.74 articles"]

        EC2["📐 Centroid Drift<br/>(thread → general news)"]
        EC2 --> S2["Dynamic EMA: alpha = 0.1/ln(n+2)<br/>At 50 members, each article<br/>shifts centroid only 2.5%<br/><br/>3 off-topic articles = ~7.5% shift<br/>→ not enough to change matching"]

        EC3["📰 Roundup Pollution<br/>(Market Talk matches everything)"]
        EC3 --> S3["Filtered at RSS ingest<br/>(1_wsj_ingest.py)<br/>Never enters pipeline"]

        EC4["🤖 LLM Garbage Groups<br/>(e.g. 'China EV' + 'China Earthquake')"]
        EC4 --> S4["Embedding cross-validation:<br/>avg pairwise must be ≥ 0.60<br/>any pair below 0.25 → rejected<br/><br/>'China EV' vs 'China Earthquake'<br/>= ~0.35 → group rejected"]
    end

    subgraph LIFECYCLE ["Thread Lifecycle — Splitting, Merging, Aging"]
        direction LR
        EC5["👯 Near-Duplicate Threads<br/>(LLM creates thread that<br/>already exists)"]
        EC5 --> S5["Merge check: new centroid vs<br/>all existing threads > 0.92 → merge<br/><br/>'Federal Reserve Rate Decision'<br/>vs 'Fed Holds Rates at 4.5%'<br/>= 0.93 → merged"]

        EC6["🔫 Same Topic, Diff Events<br/>(e.g. two separate shootings)"]
        EC6 --> S6["Time penalty separates them:<br/>'Texas Shooting' last article 10d ago<br/>threshold = 0.73 + 0.10 + size ≈ 0.87<br/><br/>'Ohio Shooting' at 0.76 similarity<br/>→ doesn't pass → new thread"]

        EC7["📈 Story Evolution<br/>(trade war → tariffs<br/>vs agriculture)"]
        EC7 --> S7["Sub-stories naturally split:<br/>as focus diverges, embeddings<br/>move apart → new articles fail<br/>to match old broad centroid<br/>→ LLM creates focused threads"]

        EC8["⏳ Long-Running Story<br/>(6-month saga)"]
        EC8 --> S8["Centroid follows slowly (low alpha)<br/>Gradual shift → thread adapts<br/>Dramatic shift → new articles<br/>won't match → new thread forms"]

        EC9["🧟 Resurrection<br/>(quiet story re-erupts)"]
        EC9 --> S9["Archived thread (14+ days inactive)<br/>still compared during matching<br/>New article matches → sets active=true<br/>→ re-appears in feed"]
    end

    subgraph VOLUME ["Volume & Assignment"]
        direction LR
        EC10["💥 Volume Spike<br/>(50 articles/day<br/>during market crash)"]
        EC10 --> S10["Centroid matching scales linearly<br/>First articles create threads via LLM<br/>Later articles match established threads<br/>Anti-gravity prevents black hole"]

        EC11["↔️ Article Fits 2 Threads<br/>('Fed impact on housing')"]
        EC11 --> S11["Margin check: if best - 2nd < 0.03<br/>→ ambiguous → left unmatched<br/><br/>e.g. 0.81 vs 0.79 = 0.02 margin<br/>→ unmatched → LLM decides<br/>(1:1 FK: one thread per article)"]

        EC12["🆕 New Topic Appears<br/>(no existing thread matches)"]
        EC12 --> S12["Fails all centroid matching<br/>→ enters LLM grouping<br/>→ if 2+ articles about same topic<br/>   Gemini creates new thread<br/>→ if unique, stays standalone"]
    end
```

## 9. Algorithm Constants

```mermaid
mindmap
  root((Threading<br/>Constants))
    Matching
      Base threshold: 0.73
      Time penalty: 0.01/day
      Size penalty: 0.04 × ln(n+1)
      Hard cap: 50 members → 0.87
      Match margin: 0.03
    Centroid
      EMA alpha: 0.1 / ln(n+2)
    LLM Grouping
      Min avg similarity: 0.60
      Min pair floor: 0.25
      Max group size: 8
      Daily cutoff: 2 days
    Merge
      Threshold: 0.92
    Lifecycle
      Cooling: 3+ days
      Archive: 14+ days
    Heat Score
      must_read weight: 3
      worth_reading weight: 2
      optional weight: 1
      Decay: e^(-0.3 × days)
```
