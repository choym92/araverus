## Key Decisions & Rationale

| Decision | Rationale |
|----------|-----------|
| **FilterPanel** (slide-in vs dropdown) | Better for persistent filtering; no overlay on desktop = content remains scannable; matches Sidebar pattern |
| **Tab typography-only** | Editorial hierarchy; pure weight/size vs background decoration; aligns with 2026 "barely-there UI" trend |
| **2-column related cards** | Rich visual context (image + summary); mirrors main feed featured/standard patterns; similarity bar removed (data too clustered 72-74%) |
| **Breadcrumb over back arrow** | Navigation context (not just escape); clickable category links; editorial standard |
| **Keywords centered + hashtag** | Micro-interactions (spring scale) feel modern; hashtags clarify they're tags; centered = focal point after image |
| **Summary paragraphs** | Wall-of-text problem solved; lead + grouped body = breathing room; `max-w-2xl` = optimal reading width |

## Blockers / Questions

- **RPC Enrichment Cost**: 2-phase fetch for related articles adds latency. Consider:
  - Cache related articles at page render time (unstable_cache)
  - Or: Extend RPC to return additional fields directly (SQL-level join)
- **Similarity Bar Removed**: User said "data too clustered" (72-74% range). But it's the only quantitative relevance signal. Could revisit with:
  - Percentile ranking instead (1st / 2nd / 3rd most similar)
  - Or: Subtle visual (filled dot ●●● vs ●● vs ●) instead of bar
- **Image Loading**: RelatedCards now Client-side (useState for imageError). Watch bundle size if many cards rendered.

## Key Patterns & Gotchas

1. **Typography-Only Hierarchy**: No borders/backgrounds. Entirely weight/size/color. Works for 2 levels; avoid 3+ without visual separation.
2. **Centered Content Shift**: `lg:pr-72` on both nav AND main content ensures sticky nav doesn't desync from content.
3. **Mobile Tap Targets**: All clickable keywords/pills should be ≥44px height. Used `py-2.5` (~44px total) for filter panel keywords.
4. **Enrich Map Pattern**: When fetching related data, build a Map for O(1) lookup during transform. Safer than assuming order.
5. **Graceful Image Fallback**: `useState(false)` for imageError; fallback layout doesn't reflow. Check `onError` callback is attached.
